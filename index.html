從 obsidian 寫作到發佈

發表於 2023-09-21

前一陣子先是迷上了使用靜態頁面產生器創建了自己的部落格，接著想串接目前在使用的筆記軟體進行寫作，整個過程有一定的雖不困難，但細瑣的步驟整理在此，供需要的朋友使用。

選擇文字編輯器

首先選擇你喜歡的文字編輯器，選擇要點就自己看的順眼最重要。單單這一向我所使用過的文字編輯器太多種，目前定居在 obsidian 主要的原因還是因為它漂亮，在寫作時最能讓我有滿滿的沉浸感。

使用 vscode 運行 hexo

這個動作最主要的目的是將自己的文章渲染成靜態網頁，安裝 vscode 的好處一部分原因也是當你想為自己的部落格做一點”硬體”上的裝修與改變時，可以透過一點簡單的程式碼修改完成。

佈署至 GitHub Pages

GitHub Pages 是 GitHub 所提供的網頁托管服務，在自己的流量還沒有大到需要獨立門戶，是很棒的免費服務，對於非商業用途的個人網誌而言，非常非常夠用了。

操作流程

  • 在 obsidian 內完成寫作
  • 透過 Markdown Blogger Plugin 的 push 功能將文章推送本地的部落格文章資料夾內
  • 在終端機內運行 hexo serve
  • 在瀏覽器預覽畫面
  • 如需修改錯字、排版，於 obsidian 內修改後再次推送
  • 清除快取並執行檔案編譯： hexo clean && hexo g
  • 佈署至 GitHub Pages ： hexo deploy

如何使用標籤管理上萬則卡片筆記？

整理的大前提「不是分類」，而是「可以搜尋關鍵字找到」。

什麼時候不需要使用『標籤』？

如果有明確的關鍵字可以搜尋的到，就沒有必要另外下『標籤』，每一個分類動作都是多餘的浪費時間。只是花時間做出好看的整理而已。

用標籤安排進度、使用需求：

要用標籤去管理那些無法被搜尋到的需求，例如：進度安排、使用情境安排。這樣的需求才不容易被搜尋的到，也才會常被拿出來使用。

進度安排：

『初步想法的寫作』：可能只有一個想法，還不確定是否要寫？要如何寫？

『進行中的寫作』：我寫到一半的文章，可能資料蒐集的還不夠齊全，有些想法還序要驗證。

『最後一哩路的寫作』：該有的論述、資料都有了，只差一段完整的時間統整好即可發佈。

情境安排：

『五分鐘的空檔』：一些可以隨手處理好的資料、待閱讀的短文

『二十分鐘的空檔』：記錄一些想法、規劃一些待辦事項

『一小時的空檔』：可以讀一些資料、產出一些文字。

Esor 的常用標籤：

在「工作、生活中的目標專案筆記」上面

  • 每日推進

  • 企劃醞釀

  • 定期追蹤

一直持續進行的「電腦玩物寫作目標」

  • 撰寫完成

    • 進度超過 50% ，並且想法已經抵定，就是要花時間最後寫完的文章。

    • 當我有一些比較正式的專注時間時，我會打開「撰寫完成」標籤，推進這些即將寫完的文章進度。

  • 測試研究

    • 需要用一些數位裝置測試的工具，需要實踐一下的方法。

    • 當我有一些零碎時間，但腦筋轉不動時，我會打開「測試研究」標籤，推進這些需要先測試看看（動動手就好，不一定要動腦）的文章進度。

    • 測試到一定階段，這些文章就可以轉換成「撰寫完成」標籤。

  • 思考動腦

    • 需要想清楚方法論，需要解析問題的文章題目。

    • 當我有一些零碎時間，而且腦筋可以轉動，我會打開「思考動腦」標籤，推進這些需要動動腦的題目。

    • 解出一定程度答案，這些文章就可以轉成「撰寫完成」標籤

『針對生活中的任務』，想要執行的興趣、想要學習的技能的筆記：

  • 兩人時間

    • 有些事情是可以跟另一半一起完成的，這樣的筆記我就加上這個標籤，當出現適合的空檔時，打開這個標籤，就可以選擇想要推進的任務。

  • 一人空閒

    • 有些興趣只有我自己有興趣，例如我自己想讀的書，我就加上這個標籤，當出現我一個人的空檔時（例如上完課回程的高鐵上），我就打開這個標籤，選擇一件事情來推進。

  • 想和孩子一起做

    • 有些娛樂、學習是可以和孩子一起做的，我就把這樣的筆記加上這個標籤，現在你應該知道我什麼時候會打開了吧？

結論

真正好用的「標籤（Tag）」，可以告訴我在目前的空檔時間中，我可以採取什麼更好的行動。

真正好用的「標籤（Tag）」，應該是一種真正有效的標注，並且跟著任務筆記的執行狀態而改變，從而幫助我追蹤需要的執行狀態。

Hexo 部落格用法指南

發表於 2023-08-31 更新於 2023-09-08 分類於 大分類 ， 小分類 ， 小分類

Hexo 部落格

清快取 接著執行 編譯加佈署

hexo clean && hexo g –d

快速啟動

建立新文章

$ hexo new article-name-here

啟動本地伺服器

$ hexo serve

指定 port

hexo s -p 4001

編譯靜態網頁

$ hexo generate

佈署至 github (需先設定好config)

$ hexo deploy

寫作

建立草稿

hexo new draft new-draft-name

建立文章

hexo new post new-post-name

建立頁面 http://localhost:4000/new-page-name/

hexo new page new-page-name

檔名變更 （ new_post_name setting in _config.yml ）

:year-:month-:day-:title.md

將草稿匣的文章移動至已發佈

hexo publish post test-article

從預先搭好的鷹架建立你的文章 ( 在 scaffolds 資料夾中有預先建立好的模版 )

hexo [layout] post-name

Front-matter

特殊引用

Writing is good, writing is cool,

Author url link-title

Youtube 引用

Youtube 播放清單引用

一鍵發佈，兩種寫法

$ hexo generate –deploy
$ hexo deploy –generate

串接寫法&& 與簡寫

hexo clean && hexo g –d
