CodingDay002：CSS 基礎操作與綜合演練

發表於 2023-10-26

昨天我學到使用 html 標記網頁內容，今天學習的是使用 CSS 來將網頁內容進行排版。

  1. 首先在 <head> 內增加 <link href="styles.css" rel="stylesheet"/> 使瀏覽器知道透過參照連結去找到所屬的樣式表。

  2. 在 styles.css 內透過選擇器選取目標物

    • type selector : elementtype {}
    • id selector : #idname {}
    • class selector : .classname {}

  3. 對目標物進行樣式變更

    • 背景圖片 : background-image: url();
    • 文字對齊方式：text-align: center;
    • 寬度設定：`width: 80%;
    • 底色設定：background-color: burlywood;
    • 留邊設定：`margin-left: auto;

  4. 綜合演練： 在同一列顯示品名與價格

    • 將每一個品項以 <article> 群組化，並設定其 classname=item
    • 將單一品項的產品名稱、價格以 <p class=product> <p class=price> 命名
    • 透過選擇器選取 .item p {display: inline-block;}
    • 顯示為行內元素：display: inline-block;
    • 寬度設定：width: 50%;

CodingDay001：寫紙本筆記

發表於 2023-10-25

HTML 為你的網頁增添內容

之前就有學習過 HTML ，所以這個單元很快就通關，比較多都是複習語法與熟悉全英文的教學介面。總計大約二小時，除了複習標題、內容、段落的標記語法，也更熟悉元素(element)、屬性(attribute)、值(value)之間的關係與使用方式。

HTML(HyperTextMarkupLanguage)超文本標記語言，簡單來說就是把你的網頁內容透過標記的方式，使得瀏覽器可以讀懂你這段是文字、圖片、影片，配上低配版的排版功能，得以呈現出一個內容為主，卻不怎麼美觀的網頁。

寫紙本筆記：少剪貼，多用腦

學習的過程之中最好準備一本紙本的筆記本(就算有用筆記軟體也建議再準備一本紙本筆記)，把常用到的重點指令、或容易混淆的觀念記錄下來。除了印象更加深刻、方便自己隨時查閱使用以外，最大的好處是強迫你不能無腦剪貼筆記，需用自己的方式整理內容。

CodingDay000：練習是一切的終極答案

發表於 2023-10-25

偶然發現到一個超棒的 coding 自學網站 FreeCodeCamp，是那種真希望我十年前就遇到它的那種棒。除了教材安排的難度很適合初學者，學習中的回饋機制也很棒，每一個小步驟通關都會有慶祝小語、數十個小步驟可以完成一個小專案、幾個小專案則可以獲得能在 LinkedIn 曬圖的證書。

雖有多次中斷學習的失敗經驗，但最終我還是決定再一次重新學習編程，這次的學習沒有很強的目的性，也沒有想要轉職的意圖，作為興趣愛好的學習成分更高了一點，也想要透過學習編程讓自己在工作之餘有一個可以持續研究的主題，一個部落格寫作的素材。當然，也許就只是單純喜歡自己動手製作產品的成就感。

有關編程，我總是以為那是很吃天賦的才能，或者至少要身處能持續編程的環境。剛好昨天看的一個 talk # What I Gained After 100 Days of Code講到這件事。如果問成功的自學者有什麼秘訣？簡言之練習就是一切的終極答案，遠超過訣竅、天賦、秘徑等等等。

也因此我打算每天排一小時專心練習編程。

預計使用 FreeCodeCamp 這個網站的證書任務來循序漸進的練習編程。並且每天把學習心得彙整寫成部落格文章。

這是一篇如何寫出有效文章的教學文。

我寫每一篇文章都要花很多時間，但卻沒有感覺有明顯的進步。有別於過往是從一片白紙的狀態開始『自由書寫』，接下來我想嘗試『刻意練習』的方式強化我對寫作品質的掌握程度。而本次使用教材是洪震宇的『精準寫作』。

起承轉合的寫作思維會妨礙讀者理解，在寫作者與讀者間設立了一道道的阻礙，使得讀者得像撥洋蔥一樣，一層一層的撥開起承轉合的路線導引，才能抵達寫作者想傳達的觀點與結論。

寫作不必如此嬌羞。每一段的第一句就直球破題，勇敢的寫出自己的觀點。把你最重要的壓箱寶直接端上桌。

必要的脈絡資訊再透過接續段落補述即可。

執行要點

  • 使用 Bullet point 的方式條列出要點
  • 使用 Hierarchy 的架構組織文章結構
  • 每個段落的 Punch 在第一句就進場
  • 使用 ROA 架構文章骨幹
    • 設定 Reader property 提醒自己讀者是誰
    • 設定 Object property 自己目的為何
    • 設定 Action property 確認引導的行動
  • 用自己的方式寫出 Memorizable 的結論

我使用 Obsidian 寫部落格

發表於 2023-09-21 更新於 2023-09-27

使用 Obsidian 寫作的工作模式

隨著在 Obsidian 的筆記內容逐漸增加，逐漸的將概念較為完整的筆記改寫成部落格文章，讓每一篇文章為自己帶進各式各樣流量、業務、合作機會。透過 Hexo 一鍵完成網站的擴建，佈署至位於 GitHub Pages 的所在地，任何人在任何地方在任何時間都可以透過一個網址造訪你的網站。而這一切的運作只需要一台可以連網的電腦。這也是我正在運作的工作模式。

減輕非核心工作的負擔

過去使用 WordPress 架站的我在今年投向了 Hexo 的懷抱。主要原因是我更想專注在文字創作，而不是花費過多的時間調整版型、探索功能、軟體更新等許多非核心工作的工作。

在了解到使用靜態頁面的好處(快速、安全、便宜、自主權)後，花了一些時間學習使用靜態頁面產生器創建自己的部落格，串接一直在使用的筆記軟體 Obsidian 進行寫作，整個流程建置與工具選擇花了不少的心力，總算調配出一個比較適合我的工作流程。將其結論整理在此，做個記錄，方便自己檢索，也供他人參考。

為什麼選擇 Obsidian

首先選擇你喜歡的文字編輯器，我選擇 Obsidian 要點有三。

  1. 介面的沉浸感
  2. 檔案自主權
  3. 使用成本

首先，我使用過至少十種以上的文字編輯器，目前最喜歡的還是黑曜石 Obsidian ，除了名字好聽、介面簡潔，還可以安裝有趣實用的外掛，訂製專屬的個人化設定，更重要的是在寫作時讓我有滿滿的沉浸感，不容易分心去做其他事。

第二，寫作文檔以 markdown 的格式保存在自己的電腦，可以自己選擇備份的方式，也保留日後若有想要搬移部落格的彈性，所有的檔案都是以兼容性的最高的形式存在。

第三，不用被 SasS 服務各種綁架，我不喜歡檔案強制上傳被收取使用費的感覺。也不喜歡按月收費而不是按實際使用量收費。在還沒這麼確定自己可以大量、持續寫作前，Obsidian 是免費陪著我一起前進的文字編輯器。

將文章編譯成網頁

我們寫好的文章如果未經處理，就無法依照各個不同的閱讀載具，調整成最適合的閱讀格式，送到讀者眼中。所以我們要先透過靜態網站產生器 (Static Site Generator) 將文章預先處理成瀏覽器所能讀取的檔案格式 html (Hypertext Markup Language) 。

這邊我選擇的是 Hexo 靜態網站產生器。

這個動作最主要的目的是將自己的文章編譯成靜態網頁，Hexo 速度很快，教學資源也多，而我當初挑選它最主要的原因是我的老 Mac 對 node.js 的相容較好，不像一些比較晚出現的 SSG 在編譯時一直會跳出 macOS 版本過舊的提示訊息。我就只是想要寫文章，不要強制我要一直更新那些我根本用不到的功能。

編譯的執行需要透過終端機下指令，可以透過內建的 terminal.app ，我則是選擇安裝 vscode 編輯器，除了可以進行 CLI (Command Line Interface) 指令的執行，更可以在此編修自己網站的程式碼，為自己的部落格做一點”硬體”上的裝修與改變。

將網頁佈署至 GitHub Pages

我們所寫的每一篇文章就像是公司招募進來的員工，每個員工都有其擅長的項目，為你的公司帶來關注與業務。而編譯好的網站就是所有員工上班的辦公室。辦公室還需要一塊土地與門牌號碼。

GitHub Pages 是 GitHub 所提供的網頁托管服務，在自己的流量還沒有大到需要獨立門戶，是很棒的免費服務，對於非商業用途的個人網誌而言，非常非常夠用了。更提供一組以你註冊 GitHub 帳號的專屬網址。例如：yourname.github.io。就幫你一併完成了辦公室門牌號碼的的申請，方便他人透過這個網址造訪你的辦公室。

辦公室所身處的土地就是伺服器托管，也就是你編譯好的程式碼要放在哪一台電腦裡，因為家用電腦可能會關機、連線不穩等等的因素，所以會把網站放在專門把電腦租給他人托管網站的伺服器廠商，常見的有 Netify、Vercel 等等，這邊我選擇的是使用 GitHub Pages，除了免費托管以外，可以一併解決土地與門牌號碼的問題。讓我在還不知道自己能持續寫多久之前，先專心的把精力集中在寫作本身，而不用花太多心神維護各種工具。

操作流程

綜合以上，濃縮成以下步驟

  • 在 Obsidian 內完成寫作
  • 透過 Markdown Blogger Plugin 的 push 功能將文章推送本地的部落格文章資料夾內
  • 在終端機內運行 hexo serve
  • 在瀏覽器預覽畫面
  • 如需修改錯字、排版，於 obsidian 內修改後再次推送
  • 清除快取並執行檔案編譯： hexo clean && hexo g
  • 佈署至 GitHub Pages ： hexo deploy

結論

雖然我不是電腦工程師，但整個寫作過程搭配使用的工具套組讓我感覺自己就是個電腦工程師，也很開心自己多掌握了一套技能樹。
