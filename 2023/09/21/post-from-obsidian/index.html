用寫作建立你的商業帝國

發表於 2023-09-21 更新於 2023-09-24

願景

隨著你的寫作內容逐漸增加，每一篇文字員工為你帶進各式各樣流量、業務、合作機會，你透過 Hexo 一鍵完成辦公室的擴建，佈署至位於 GitHub Pages 的總部所在地，任何人在任何地方在任何時間都可以透過你的網址造訪你的辦公室。而這一切的運作只需要一台可以連網的電腦。這也是我正在追求的辦公模式。

過去使用 WordPress 架站的我投向了 Hexo 的懷抱。在今年改用靜態頁面產生器創建自己的部落格，串接一直在使用的筆記軟體 obsidian 進行寫作，整個流程建置與工具選擇花了不少的心力，總算調配出一個比較適合我的工作流程。將其結論整理在此，做個記錄，方便自己檢索，也供他人參考。

選擇文字編輯器

首先選擇你喜歡的文字編輯器，我的選擇要點有三。

  1. 檔案自主權
  2. 介面的沉浸感
  3. 使用成本

我使用過至少十種以上的文字編輯器，目前最喜歡的是黑曜石 obsidian，除了名字好聽、介面簡潔，在寫作時讓我有滿滿的沉浸感以外，也可以安裝有趣的外掛裝潢一下的文字編輯器，打造專屬於你自己的個人化設定。但最主要的原因是可以將檔案以 markdown 的格式保存在自己的電腦，不用被綁架一定什麼都要上雲端，不用擔心用習慣以後就只能乖乖付費。

將文章編譯成網頁原始碼

我們寫好的文章如果未經處理，就無法依照各個不同的閱讀載具調整成最適合的閱讀格式。所以我們要先透過靜態網站產生器 (Static Site Generator) 將文章預先處理成瀏覽器所能讀取的檔案格式 html (Hypertext Markup Language) 。

這邊我選擇的是 Hexo 靜態網站產生器。

這個動作最主要的目的是將自己的文章編譯成靜態網頁，Hexo 速度很快，教學資源也多，而我當初挑選它最主要的原因是我的老 Mac 對 node.js 的相容較好，不像一些比較晚出現的 SSG 在編譯時一直會跳出 macOS 版本過舊的提示訊息。我就只是想要寫文章，不要強制我要一直更新那些我根本用不到的功能。

編譯的執行需要透過終端機下指令，可以透過內建的 terminal.app ，我則是選擇安裝 vscode 編輯器，除了可以進行 CLI (Command Line Interface) 指令的執行，更可以在此編修自己網站的程式碼，為自己的部落格做一點”硬體”上的裝修與改變。

佈署至 GitHub Pages

我們所寫的每一篇文章就像是公司招募進來的員工，每個員工都有其擅長的項目，為你的公司帶來關注與業務。而編譯好的網站就是所有員工上班的辦公室。辦公室還需要一塊土地與門牌號碼。

GitHub Pages 是 GitHub 所提供的網頁托管服務，在自己的流量還沒有大到需要獨立門戶，是很棒的免費服務，對於非商業用途的個人網誌而言，非常非常夠用了。更提供一組以你註冊 GitHub 帳號的專屬網址。例如：yourname.github.io。就幫你一併完成了辦公室門牌號碼的的申請，方便他人透過這個網址造訪你的辦公室。

辦公室所身處的土地就是伺服器托管，也就是你編譯好的程式碼要放在哪一台電腦裡，因為家用電腦可能會關機、連線不穩等等的因素，所以會把網站放在專門把電腦租給他人托管網站的伺服器廠商，常見的有 Netify、Vercel 等等，這邊我選擇的是使用 GitHub Pages，除了免費托管以外，可以一併解決土地與門牌號碼的問題。讓我在還不知道自己能持續寫多久之前，先專心的把經歷集中在寫作本身，而不用花太多心神維護各種工具。

操作流程

  • 在 Obsidian 內完成寫作
  • 透過 Markdown Blogger Plugin 的 push 功能將文章推送本地的部落格文章資料夾內
  • 在終端機內運行 hexo serve
  • 在瀏覽器預覽畫面
  • 如需修改錯字、排版，於 obsidian 內修改後再次推送
  • 清除快取並執行檔案編譯： hexo clean && hexo g
  • 佈署至 GitHub Pages ： hexo deploy
